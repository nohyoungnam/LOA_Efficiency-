위 내용들은 add_icon - class 안에 있는 이미지들을 설명한 내용이야 안에 mapping.csv 있을건데 이거 읽고
각 클래스에 맞게 아이콘들을 lostark_skill_db_output여기 안에 클래스 별로 나뉘어 져있을건데 복사 붙여 넣어서 잘 들어가게 해줬으면 좋겠어 스킬아이콘으로 대조하고 이름을 찾는건 lostark_skill_db_output - 클래스 폴더안에서 찾겠끔. api에서 해당 캐릭터가 무슨 클래스인지 아니깐 이렇게 범위를 더 좁혀서 찾게 하면 더 빠르게 읽혀 질거야

add_icon/class/ 읽히도록 스캔 설정
class 폴더 안 각 클래스 이름을 읽고 api에서 읽은 같은 클래스의 폴더 읽히도록 설정

add_icon 폴더 안에 있는 내가 설명하는 이미지들은 클래스 상관없이 모두 쓰는 아이콘
etc_attack.png = 기타
basic_attack.png = 기본 공격
Bleed rune.png = 스킬룬 출혈
Poison rune = 스킬룬 중독
현재 위 4개의 모든 클래스가 쓰는 아이콘은 폴더 하나 만들어서 따로 모으는게 좋을거 같아

battle time.jpg 이건 전투분석기 읽을때만 쓰는거니깐 이것도 공용으로 들어가면 좋을거 같아

lostark_skill_db_output 폴더에는 현재 모든 직업의 스킬 아이콘 이미지가 있어
all_skill_name_map.csv 파일을 열어보면 모든 클래스의 스킬명 그리고 스킬파일의 이미지 이름이 적혀져 있어
현재 스킬아이콘 이미지를 api를 통해 불러와서 매칭하는 방식인데
이제는 lostark_skill_db_output에서 각직업별 폴더로 나눠 놨으니
api를 불러오고 해당 캐릭터의 클래스를 알거 잖아 그러면 해당 클래스의 폴더를 찾아가서 거기서 이미지를 가져와서 비교하는 거지 그러면 더 빠르게 될거 같아

그리고 ocr에서 이미지 컷팅하는것도 지금 컷팅하고 나누는게 최적화가 되어 있는지 확인이 필요로해
속도를 더 빠르게 인식할 수 있는 방법을 찾으려고 해
